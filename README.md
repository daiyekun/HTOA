## 系统概述

CWHT OA 系统是一套功能完善的企业办公自动化平台，参考致远互联 A8 和泛微 e-cology 设计理念打造。系统采用前后端分离架构，后端基于 .NET 10 框架，采用 DDD（领域驱动设计）架构模式，前端使用 Vue 3 + TypeScript + Ant Design Vue 4。

### 技术架构

| 层级 | 技术栈 |
|------|--------|
| 后端框架 | .NET 10 |
| ORM | FreeSql |
| 数据库 | PostgreSQL 16 |
| 缓存 | Redis |
| 前端框架 | Vue 3.5 + TypeScript |
| UI组件库 | Ant Design Vue 4 |
| 状态管理 | Pinia |
| 路由 | Vue Router 4 |
| 构建工具 | Vite 8 |

### 二、工作流引擎

#### 2.1 流程设计器（核心功能）

**拖拽式流程设计**
- 可视化拖拽界面
- 多种节点类型支持
- 连线自动吸附
- 缩放与画布拖拽

**节点类型**

| 节点类型 | 功能说明 | 配置项 |
|---------|---------|--------|
| 🔘 开始 | 流程起点 | 触发事件 |
| ⏹️ 结束 | 流程终点 | 结束动作 |
| 👤 审批 | 人工审批节点 | 审批人、审批方式、超时设置 |
| 🔀 条件分支 | 条件判断分流 | 分支条件、默认分支 |
| ⚡ 并行 | 并行执行/汇聚 | 分支数量、汇聚方式 |
| 📧 抄送 | 通知节点 | 抄送人、抄送时机 |
| 📦 子流程 | 调用子流程 | 子流程选择 |
| ⚙️ 服务调用 | HTTP服务调用 | 服务地址、请求方式 |

**审批方式**

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 或签 | 任意一人审批通过即可 | 快速审批 |
| 会签 | 所有人审批通过才算通过 | 重要决策 |
| 顺序签 | 按顺序逐个审批 | 逐级审批 |
| 抢签 | 任意一人先审批即结束 | 任务分发 |

**审批人类型**
- 指定人员
- 角色
- 部门负责人
- 部门分管领导
- 发起人自选
- 发起人自己
- 上级领导
- 连续上级

**节点高级配置**
- 超时设置（自动通过/自动拒绝/通知）
- 转办/委派/打回设置
- 审批意见必填
- 多人审批规则

**连线功能**
- 拖拽连线
- 条件表达式设置
- 优先级配置
- 分支颜色区分
- 连线点击编辑/删除

#### 2.2 流程定义管理

- 流程版本管理
- 流程启用/禁用
- 流程复制

#### 2.3 流程实例管理

## 快速开始

### 环境要求

- Node.js 18+
- .NET 10 SDK
- PostgreSQL 16+
- Redis 7+

### 启动步骤

#### 1. 启动数据库

```bash
docker-compose up -d
```

#### 2. 运行迁移

```bash
cd backend/src/CWHT.OA.Migration
dotnet run
```

#### 3. 初始化数据

```bash
cd backend/src/CWHT.OA.Seed
dotnet run
```

#### 4. 启动后端

```bash
cd backend/src/CWHT.OA.WebAPI
dotnet run
```

#### 5. 启动前端

```bash
cd frontend
npm install
npm run dev
```

### 访问地址

- 前端: http://localhost:5173
- 后端API: http://localhost:5000
- 测试账号: admin / admin123
