## 二、流程设计器详细设计 (重点)

### 2.1 节点类型

| 节点类型 | 图标 | 说明 | 配置项 |
|---------|------|------|--------|
| 开始 | ▶️ | 流程起点 | 名称、触发事件 |
| 结束 | ⏹️ | 流程终点 | 名称、结束动作 |
| 审批 | 👤 | 人工审批 | 审批人、审批方式、超时设置、打回设置 |
| 条件 | 🔀 | 条件分支 | 分支条件、默认分支 |
| 并行 | ⚡ | 并行执行 | 分支数量、汇聚方式 |
| 子流程 | 📦 | 调用子流程 | 子流程选择、参数映射 |
| 抄送 | 📧 | 通知抄送 | 抄送人、抄送时机 |
| 服务 | ⚙️ | 服务调用 | 服务地址、请求方式 |

### 2.2 审批方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 或签 | 任意一人审批通过即可 | 快速审批 |
| 会签 | 所有人审批通过才算通过 | 重要决策 |
| 顺序签 | 按顺序逐个审批 | 逐级审批 |
| 抢签 | 任意一人先审批即结束 | 任务分发 |

### 2.3 审批人类型

| 类型 | 说明 |
|------|------|
| 指定人员 | 固定人员审批 |
| 角色 | 按角色分配 |
| 部门负责人 | 发起人所在部门负责人 |
| 部门分管领导 | 发起人所在部门分管领导 |
| 发起人自选 | 发起人自己选择 |
| 发起人自己 | 发起人自己审批(无需审批) |
| 上级领导 | 发起人的上级 |
| 连续上级 | 逐级向上直到有审批权限者 |

### 2.4 连线功能

- **拖拽连接**: 从节点右侧拖出连线到下一节点左侧
- **条件设置**: 点击连线设置条件表达式
- **分支颜色**: 不同条件分支显示不同颜色
- **删除连线**: 选中连线后按Delete删除

### 2.5 表单设计器 (后续扩展)

- 拖拽式表单组件
- 支持字段: 单行文本、多行文本、数字、日期、下拉单选、下拉多选、附件、用户选择、部门选择
- 表单校验规则
- 公式计算

## 四、API设计规范

### 4.1 请求格式
```
POST /api/{module}/{resource}
GET /api/{module}/{resource}
PUT /api/{module}/{resource}/{id}
DELETE /api/{module}/{resource}/{id}
```

### 4.2 响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "success": true
}
```

### 4.3 分页格式
```json
{
  "code": 200,
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

## 六、安全设计

### 6.1 认证
- JWT Token认证
- Token有效期: 24小时
- RefreshToken: 7天

### 6.2 权限
- 基于角色的权限控制(RBAC)
- 数据权限: 本人/本部门/本部门及以下/全部
- 接口权限: 菜单权限 + 按钮权限

### 6.3 日志
- 操作日志: 记录所有CRUD操作
- 登录日志: 记录登录登出
- 业务日志: 记录关键业务操作
